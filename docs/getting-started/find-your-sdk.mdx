## Quick decision flow

1. If your bundler/framework exists in the table, use that SDK and guide.
2. If you are integrating into an existing app with custom structure, check [/integrations/existing-app](/integrations/existing-app).
3. If you need multi-bundler Module Federation, check [/integrations/vite-rspack-webpack-mf](/integrations/vite-rspack-webpack-mf).
4. If your stack is not listed, use `zephyr-agent` as a low-level fallback.

## Fallback for unsupported stacks

If your framework has no official SDK yet, use `uploadOutputToZephyr` from `zephyr-agent` to upload a generated output directory.

Best fit:

- custom framework adapters
- internal platform tooling
- plugin maintainers

Minimal usage:

```ts
import { uploadOutputToZephyr } from 'zephyr-agent';

await uploadOutputToZephyr({
  rootDir: process.cwd(),
  outputDir: '.output',
  ssr: true,
});
```

For full fallback behavior (options, mapping, SSR/CSR, errors), see `zephyr-agent` docs in the packages repository.
