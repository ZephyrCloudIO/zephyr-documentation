## Key Areas for Style Customization

1. **Color Scheme**: Modify CSS variables in your custom `styles/index.css` to override default colors
2. **Layout**: The `bundle.css` contains all layout styles for nav, sidebar, and content areas
3. **Typography**: Font families and sizes are controlled via CSS variables
4. **Component Styling**: Individual component styles are in the `bundle.css` with modular CSS class names
5. **Dark Mode**: Dark mode styles are defined in `.dark` selector blocks

## Recommended Approach for Patching

Since the theme files are in `node_modules`, I recommend:

1. **Continue using your custom `styles/index.css`** to override CSS variables
2. **Use the `theme/index.tsx` file** to wrap or extend components
3. **Consider creating patches** for the `@rspress/theme-default` package if deeper modifications are needed

The main styling control happens through the CSS custom properties system, which allows for extensive customization without modifying the core theme files directly.
