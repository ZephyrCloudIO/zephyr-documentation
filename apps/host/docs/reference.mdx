## API Reference

### REST API
Complete REST API documentation for programmatic access:

```bash
# Authentication
POST /api/v1/auth/login
GET /api/v1/auth/me

# Applications
GET /api/v1/applications
POST /api/v1/applications
PUT /api/v1/applications/{id}
DELETE /api/v1/applications/{id}

# Deployments
GET /api/v1/deployments
POST /api/v1/deployments
GET /api/v1/deployments/{id}

# Environments
GET /api/v1/environments
POST /api/v1/environments
PUT /api/v1/environments/{id}
```

### GraphQL API
Query deployment data and metrics:

```graphql
query GetApplications {
  applications {
    id
    name
    status
    versions {
      id
      version
      deployedAt
      environment
    }
  }
}

mutation DeployApplication($input: DeployInput!) {
  deployApplication(input: $input) {
    id
    status
    deploymentUrl
  }
}
```

## Configuration Reference

### Zephyr Configuration File
Complete configuration options for `zephyr.config.js`:

```javascript
export default {
  // Application settings
  name: 'my-app',
  version: '1.0.0',
  
  // Module Federation
  remotes: {
    'remote-app': 'https://remote-app.example.com/remoteEntry.js'
  },
  
  // Build settings
  build: {
    target: 'es2020',
    minify: true,
    sourcemap: true
  },
  
  // Deployment
  deploy: {
    provider: 'cloudflare',
    domain: 'my-app.example.com',
    environments: ['development', 'staging', 'production']
  }
}
```

### Package.json Integration
Configure Zephyr in your package.json:

```json
{
  "name": "my-micro-frontend",
  "scripts": {
    "build": "zephyr build",
    "deploy": "zephyr deploy",
    "dev": "zephyr dev"
  },
  "zephyr": {
    "framework": "react",
    "bundler": "vite"
  }
}
```

## CLI Reference

### Global Commands
```bash
# Project initialization
zephyr init [--framework] [--bundler]

# Development
zephyr dev [--port] [--host]
zephyr build [--mode] [--target]

# Deployment
zephyr deploy [--env] [--tag]
zephyr rollback [version]

# Management
zephyr env list
zephyr env switch <name>
zephyr version list
zephyr logs [--lines]

```

## Error Codes & Troubleshooting

When something goes wrong, Zephyr provides detailed error codes to help you diagnose issues:

### Error Code Categories
- **ZE10xxx** - Build and compilation errors
- **ZE20xxx** - Deployment and infrastructure errors  
- **ZE30xxx** - Runtime and module loading errors

### Common Issues
- **Module Loading Failures** - Remote modules can't be loaded
- **Build Failures** - Compilation or bundling issues
- **Deployment Errors** - Cloud provider configuration problems
- **Network Issues** - Connectivity and DNS problems

For detailed troubleshooting, see our [error code reference](/errors).
